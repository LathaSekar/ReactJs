1. Understanding Component Props

Props (Properties) are used to pass data from parent → child component.
👉 Props are read-only
Example
function Child(props) {
  return <h1>Hello {props.name}</h1>;
}

function Parent() {
  return <Child name="Latha" />;
}

 2. Setting Default Props

Default props are used when no value is passed from parent.

Example
function Child(props) {
  return <h1>Hello {props.name}</h1>;
}

Child.defaultProps = {
  name: "Guest"
};

If parent does not pass `name`, output will be:
Hello Guest

3. Sending and Getting Props

🔹 Sending Props (Parent)

<Child name="Latha" age={25} />


🔹 Receiving Props (Child)
Without destructuring
```javascript
function Child(props) {
  return (
    <div>
      <h1>{props.name}</h1>
      <p>{props.age}</p>
    </div>
  );
}

 with Destructuring
```javascript
function Child({ name, age }) {
  return (
    <div>
      <h1>{name}</h1>
      <p>{age}</p>
    </div>
  );
}

4. Props Validation

Props validation ensures correct data type is passed.

We use PropTypes.

Example

import PropTypes from "prop-types";
function Child({ name, age }) {
  return <h1>{name} - {age}</h1>;
}

Child.propTypes = {
  name: PropTypes.string,
  age: PropTypes.number
};

 Helps catch errors during development.

 5. Understanding Component State
State is used to store dynamic data inside a component.
 Unlike props, state can change
 Example (Class Component)
class Counter extends React.Component {
  state = {
    count: 0
  };

  render() {
    return <h1>{this.state.count}</h1>;
  }
}
6. Working with Component State

State is updated using `setState()`.
 Example
class Counter extends React.Component {
  state = { count: 0 };

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <button onClick={this.increment}>
        {this.state.count}
      </button>
    );
  }
}

7. State vs Props (Very Important )

| Feature    | Props           | State              |
| ---------- | --------------- | ------------------ |
| Source     | Parent          | Component itself   |
| Mutability | Read-only       | Mutable            |
| Purpose    | Pass data       | Store data         |
| Control    | Parent controls | Component controls |

8. Stateless Functional Components

Stateless components are simple functions without state.
Used only for UI rendering
Example

```javascript
function Welcome(props) {
  return <h1>Hello {props.name}</h1>;
}
Modern React
Even functional components can use state using Hooks:
syntax:
const [state,setState]=useState(initialValue);
```javascript
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

9. Creating and Managing Stateless Components
Stateless components are:
* Easy to write
* Easy to test
* Reusable

Example
function Button({ label }) {
  return <button>{label}</button>;
}
 
Full Real-Time Example
function User({ name = "Guest" }) {
  return <h1>Hello {name}</h1>;
}
function App() {
  return (
    <div>
      <User name="Latha" />
      <User />
    </div>
  );
}
