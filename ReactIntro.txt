1. The Need for ReactJS

ReactJS was created by Facebook to build **fast and interactive user interfaces** for web applications.

 Problems in Traditional Web Development

* Frequent **page reloads**
* **Complex DOM manipulation**
* Difficult to maintain large applications
* Slow UI updates

 ReactJS Solves

✔ Faster UI updates
✔ Reusable components
✔ Better performance using **Virtual DOM**
✔ Easy to maintain large applications

Example use cases:

* Social media apps
* Dashboards
* E-commerce websites
 2. Advantages / Features of ReactJS

1. Virtual DOM

React creates a **virtual copy of the DOM** and updates only changed elements.

 2. Component-Based Architecture

UI is divided into reusable components.

 3. One-way Data Binding

Data flows from **parent to child** components.

4. JSX

Allows writing **HTML inside JavaScript**.

 5. Performance

Fast updates because React compares **virtual DOM with real DOM**.

6. Reusability

Components can be reused across the application.


 3. Understanding Single Page Applications (SPA)

 What is SPA?

A Single Page Application loads only one HTML page and updates content dynamically.

 Traditional Website

User Click → Request Server → New Page Load

 SPA

User Click → JavaScript updates UI → No Page Reload
 Examples of SPA

* Gmail
* Facebook
* Netflix

Benefits:

* Faster navigation
* Better user experience
* Reduced server load

 4. Exploring Component-Based Architecture

React applications are built using components.
 Component

A small reusable piece of UI.

Example:

```jsx
function Header() {
  return <h1>Welcome</h1>;
}
```

Types of Components

1️⃣ Functional Components

```jsx
function App() {
  return <h1>Hello</h1>;
}
```

2️⃣ Class Components

```jsx
class App extends React.Component {
  render() {
    return <h1>Hello</h1>;
  }
}
```

Advantages:

* Reusable
* Easy maintenance
* Modular development

Understanding React Syntax:
React uses JSX syntax.
Example:
const element = <h1>Hello React</h1>;
const name = "Latha";
<h1>Hello {name}</h1>

React Nodes:
A React Node is anything that React can render in the UI.
If React can display it in UI → it is a React Node.
React Nodes include
JSX elements
Strings
Numbers
Arrays of nodes
Components
null, undefined
JSX Element as a React Node:
function App() {
  const element = <h1>Hello React</h1>;
  return element;
}
String as React Node:
function App() {
  return "Welcome to React";
}
Number as React Node:
function App() {
  return 100;
}
React Node using Expressions:
function App() {
  const name = "Latha";
  return <h1>Hello {name}</h1>;
}
Array of React Nodes:
function App() {
  const items = ["Apple", "Mango", "Banana"];
  return (
    <ul>
      {items.map(item => <li>{item}</li>)}
    </ul>
  );
}
Component as React Node:
function Header() {
  return <h2>Welcome User</h2>;
}

function App() {
  return <Header />;
}
NULL as reactNode:
function App() {
  return null;
}
Example of reactnodes:
function App() {

  const username = "Latha";
  const marks = 90;

  return (
    <div>
      <h1>Student: {username}</h1>
      <p>Marks: {marks}</p>
    </div>
  );
}
Here React Nodes include:
<div>
<h1>
<p>
{username}
{marks}

Advantages of Using React Nodes:
Flexible rendering
Supports Dynamic UI
Reusable UI Components
Efficient Rendering

Defining and Using Node Attributes:
className
function App() {

htmlFor

dynamic attributes:
function App() {
  const url = "logo.png";

  return <img src={url} alt="logo" />;

Element Factories:
React elements were created using element factories.
React.createElement()
const element = React.createElement(
  "h1",
  { className: "title" },
  "Hello React"
);
JSX:
<h1 className="title">Hello React</h1>

Common react Events:
| Event       | Purpose        |
| ----------- | -------------- |
| onClick     | Button click   |
| onChange    | Input change   |
| onSubmit    | Form submit    |
| onMouseOver | Mouse hover    |
| onKeyDown   | Keyboard press |
Click event:
function App() {
  function handleClick() {
    alert("Button Clicked");
  }
  return <button onClick={handleClick}>Click Me</button>;
}
